# Fuiou Pay Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace direct WeChat/Alipay purchase flow with Fuiou aggregate pay, using Fuiou's official test merchant parameters by default for chain testing.

**Architecture:** Add a focused `FuiouPay` utility for signing, payload construction, HTTP JSON calls, callback processing, WeChat JSAPI creation, and active order queries. Wire purchase creation in `public/result.php` to use Fuiou `wxPreCreate` for WeChat in-app JSAPI, Fuiou `preCreate` for browser QR/link payments, and Fuiou `commonQuery` for active status confirmation.

**Tech Stack:** PHP 8, cURL, existing SQLite `Order` model, lightweight PHP CLI test script.
